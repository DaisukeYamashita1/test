EC2にWordPressをデプロイする

作業手順

  1. AWSマネジメントコンソール上から「EC2」をクリックする
  2. ナビゲーションペインから「インスタンス」を選択し、「Elastic IP」をコピーする
  3. ターミナルエミュレータを開く　※Macの場合：ターミナル　※WIndowsの場合：Tera Term
  4. ターミナルエミュレータで以下コマンドを入力し、「EC2-key.pem」が存在することを確認
    5.                 
                $ cd ~/Downloads
                $ ls -l
  5. 以下コマンドを入力し、EC2インスタンスにSSH接続する
    6.                     
                        $ chmod 600 EC2-key.pem
                        $ ssh -I EC2-key.pem ec2-user@ElasticIPアドレス

    ※Are you sure you want to continue connecting (yes/no/[fingerprint])? yes　← 選択

  6. EC2インスタンスにSSH接続後、以下コマンドを入力する
    7.                 
                    $ sudo yum -y update
                    $ sudo yum -y install mysql
  7. AWSマネジメントコンソールから、「RDS」をクリックする
  8. ナビゲーションペインから、「データベース」を選択し、「wordpress」をクリックする
  9. 「wordpress」のエンドポイントをコピーする
  10. ターミナル上から、以下コマンドを実行する
    11.                 
                    $ mysql -h RDSエンドポイント -u wordpress -p
                    Enter password：設定したパスワードを入力
                    mysql> GRANT ALL PRIVILEGES ON wordpress.* TO wordpress;
                    mysql> FLUSH PRIVILEGES;
                    mysql> quit
                    $ sudo yum install -y httpd
                    $ sudo service httpd start
  11. AWSマネジメントコンソールから、「EC2」を開く
  12. EC2コンソールから、「Elastic IP」をコピーする
  13. ブラウザ上で「Elastic IP」にアクセスし、以下ページが表示されることを確認する

解説

APACHE：世界中で仕様されているWebサーバーソフトウェア（HTTPサーバー）

Yum：LinuxのRedHat系ディストリビューションで利用されるパッケージ管理ツール

Linuxディストリビューション：Linuxが配布される形態のこと

RHEL：Red Hat社が提供する有料のLinuxディストリビューション

Wordpress：サイト作成などを可能にするコンテンツ管理システムの１つで無料のソフトウェア

httpd：HTTPを扱うデーモン

デーモン：一度起動されると常にメモリに留まって要求に応じて動作する常駐プログラム
