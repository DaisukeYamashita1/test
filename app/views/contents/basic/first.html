クラウドサービス

インターネット経由で、デスクトップ仮想化や 共有ディスクなど、ハードウェアやインフラ機能の提供を行うサービス。

クラウド事業者のひとつ。他にもGCPやAzure等があるがAWSは最も汎用的でユーザファーストなクラウド事業者である。詳しい解説はこちら

WordPress

コンテンツ管理システム（CMS）のひとつ。ノンプログラミングでWebサイトを作成して公開することが出来るため非常に人気が高い。

AWSマネジメントコンソール

GUIベースの画面でAWSを操作出来るWebサイトのこと。AWSリソースの使用状況を確認したり、構築、削除、変更、申請など様々なアクションが行える。

GUI

IaC

AWS認定

当コースの目標

当コースでは現在主流のクラウドサービス Amazon Web Services（以下AWS）を用いて、コンテンツ管理システムであるWordPressのインフラ構築を目指します。

なぜAWSなのか？

答えはシンプルでクラウドベンダの中で最大のシェアを誇っているからです。またAWSは200を超えるサービスを提供しており、ほとんどのユーザが抱える課題に対応できます。（もちろん要件によっては他のクラウドが最善の場合もあります。）AWSは今もなおシェアを拡大しておりAWSエンジニアは今後更に必要とされます。当コースを通して多くのAWSエンジニアが世の中に排出されることを祈っております。

なぜWordPressなのか？

Webの約42%はWordPressで構築されていると言われています。この数字からWordPressの需要は高いことが見受けられます。また「WordPressのインフラ構築」というのは「インフラ界のHelloWorld」とも呼ばれています。新しいインフラ技術を試すには、インフラだけではなくその上に乗るアプリケーションも必要ですよね。（インフラとアプリは切っても切り離せない関係です。）WordPressのインフラ構築は非常に手軽のため、メインであるインフラの構築に注力することができます。

作成するシステムの構成図

最終的には以下の構成のシステムを作成します。今完全に理解する必要はありません。各章を経て徐々にシステムを構築していきます。AWSには「AWSマネジメントコンソール」と呼ばれるGUIベースの画面が準備されています。最初の内はこのAWSマネジメントコンソールを用いて構築を進めていきます。最後の方では現在のインフラ構築で主流のIaC（Infrastructure as Code）にも挑戦してみましょう。このスキルはどこのIT現場でもほぼ必須と言って良い程使われている技術です。

コースの流れ

2.AWSのアカウントを作成する、3.セキュリティ初期設定

この2章は事前準備です。しかし事前準備と言いつつ「2.セキュリティ初期設定」は非常に重要です。AWSにとってセキュリティは最重要課題として取り扱われています。またAWSのアカウントを乗っ取る事件も多発しており、乗っ取られた暁には多大な損害を被ってしまいます。そうならないためにもセキュリティの設定は必ず実施していきましょう。

4.VPCを作成してみよう〜11.ホストゾーンを作成してみよう

この11章はAWSマネジメントコンソールからAWSリソースを操作し、実際に構築を進めていきます。ボリュームのある内容となっていきますが一つ一つはシンプルなので安心して取り組んでいきましょう。

12.今まで作成したリソースを削除しよう

「4.VPCを作成してみよう〜11.ホストゾーンを作成してみよう」で作成したリソースを削除します。理由はコストを抑えるためです。AWSは従量課金制を採用しており使ったら使った分だけ金額が請求されます。非常に分かり易い課金体系になっていますがAWSリソースの使い方によっては、これが吉とも凶ともなります。不要なAWSリソースは削除して、次のIaCでの構築に備えましょう。

13.CloudFormationで自動構築してみよう

いよいよIaCを用いてAWSリソースを構築していきます。構築する内容は今までと同じですが構築するための「やり方」が異なってきます。CloudFomationはAWSサービスをコーディングで作成していきます。一度覚えてしまえば非常に汎用性の高いサービスなので頑張って身に付けましょう。

当コースを通じてどのようなスキルが身に付くのか？

  • ・AWSを用いたオーソドックスな構成のシステムを構築するスキル
  • ・AWS認定 ソリューションアーキテクト アソシエイト（AWS認定 SAA）相応の構築スキル
  • ・IaCを用いた自動構築スキル

まとめ

当コースの目標はAWSを用いてオーソドックスな構成のシステム構築が目標であり、これを構築することでAWSの基礎、AWS認定 SAA相応の構築スキル及びAWSにおけるIaCを身に付けることが可能である。

デスクトップ仮想化

デスクトップ仮想化（Virtual Desktop Infrastructure：VDI）とは、サーバ上にあるデスクトップ環境を遠隔地にあるクライアント端末に転送して利用することです。

共有ディスク

ネットワーク上の複数台のコンピュータからアクセスでき、ファイルの読み書きが可能なハードディスクの領域のことです。
ハードディスク＝PC上のデータを保存・格納しておく記憶装置

インフラ

「インフラストラクチャ（Infrastructure）」の略称で、基盤となる設備や要素を指します。具体的には、パソコンやサーバーなどのハードウェア（コンピュータ関連の機械）、ネットワークを構築するインターネット・LAN・データベース・OSなどのソフトウェア（コンピューターを動かすプログラムの総称）のことを指します。

クラウド事業者

クラウドサービスを提供する事業者。

コンテンツ管理システム（CMS）

Webサイトのコンテンツを構成するテキストや画像、デザイン・レイアウト情報（テンプレート）などを一元的に保存・管理するシステム。

GUI

グラフィカルユーザインターフェース(Graphical User Interface)の略語で、マウスやタッチパネルで操作ができる画面のことをいいます。

アプリケーション

アプリケーションソフトウェアを指す。アプリケーションソフトウェアは、特定の用途や目的のために作られた、コンピューターのソフトウエア。ワープロソフト、表計算ソフト、メールソフト、ブラウザーなどが含まれます。
